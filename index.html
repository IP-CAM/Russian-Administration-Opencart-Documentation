Обновление

Инструкция ниже, предназначена для обновления магазина с одной версии на другую, например с 2.0.1.0 на 2.3.2.0

Для обновления с версии исправлений rs на следующую версию rs в пределах одной версии движка (например с 2.0.1.1 rs3 на 2.0.1.1 rs4), достаточно просто обновить файлы которые указаны в истории изменений. Или скопировать в корень сайта архив с измененными файлами, если таковой имеется (обычно он называется update-rs-to-rs).

Во многих случаях, простое обновление файлов, тоже рекомендовано для обновлении с версии на версию, но об этом предварительно читайте в теме соответствующей версии.

- Резервное копирование базы данных через ваш магазин Меню - Система - Инструменты - Импорт / Экспорт
(в версии < 2.1 Меню - Инструменты - Бекап / Восстановление).

- Желательно резервную копию базы данных делать через программу управления БД - phpmyadmin , которая установлена на любом хостинге. Это вас избавит от возможных проблем с восстановлением БД в случае удаления таблиц.

- Резервное копирование ваших файлов с помощью FTP или с помощью файлового менеджера на хостинге создать zip для всех существующих файлов и папок магазина

ВНИМАНИЕ!

6. Модификаторы

Модификаторы - специальные файлы xml в которых есть инструкции по изменению файлов. Но сами файлы не меняются, меняются их виртуальные копии. Более подробно описано в руководстве разработчика.

Но для начала совсем не об этом. Бывают случаи когда вы то и дело изменяете основные файлы, а изменения не происходят. Добавление товара

Для того чтобы добавить товар, необходимо перейти в Меню - Каталог - Товары и нажать кнопку Добавить

Далее мы увидим форму с множеством вкладок.
Каждую вкладку опишем ниже, а для того чтобы товар возможно было сохранить, необходимо заполнить обязательные поля такие как:

Название товара и Мета-тег Title на вкладке Основное
Модель на вкладке Данные
Показывать в категориях на вкладке Связи

Описание вкладок:

Основное - тут расположены основная информация о товаре. Обратите внимание на поле Мета-тег Title, это заголовок товара который будет виден вверху браузера.
Данные - на данной вкладке вводится информация о характеристиках товара, таких как: основное изображение, цена, модель, стоимость итд.
Связи - на данной вкладке задаются связи товара. Например, товар можно "связать" к определенной категории, или связать с фильтром. Для заполнения полей, предусмотрено Автозаполнение
Для добавления связи, необходимо в нужном поле ввести название элемента, например название категории. Как только вы введете первые буквы, вам предоставится выбор элементов, который можно выбрать кликнув мышью.

Основное - тут расположены основная информация о товаре. Обратите внимание на поле Мета-тег Title, это заголовок товара который будет виден вверху браузера.
Данные - на данной вкладке вводится информация о характеристиках товара, таких как: основное изображение, цена, модель, стоимость итд.
Связи - на данной вкладке задаются связи товара.
Например, товар можно "связать" к определенной категории, или связать с фильтром. Для заполнения полей, предусмотрено Автозаполнение
Для добавления связи, необходимо в нужном поле ввести название элемента, например название категории.
Как только вы введете первые буквы, вам предоставится выбор элементов, который можно выбрать кликнув мышью.

Атрибуты и группы атрибутов​

Атрибут товара это часть характеристик.
Например у товара Телефон есть атрибуты, такие как, Размер экрана, Тип батареи итд и его значения 320*480 и Li-ion

Группа атрибутов - группа характеристик, например можно разделить на группы Характеристики, Комплектность и тд


Итак, для того чтобы создать характеристики для товара, необходимо:

1. Создать группу атрибутов. Меню - Каталог - Атрибуты - Группа Атрибутов

1.1. Нажимаем Добавить и заполняем данными

2. Создать атрибут. Меню - Каталог - Атрибуты - Атрибуты

2.1. Нажимаем Добавить и заполняем данными, при это не забываем указать группу и порядок сортировки

3. Добавить атрибут к товару. Меню - Каталог - Товар

3.1. Нажимаем Редактировать и на вкладке Атрибуты добавляем наши атрибуты и вводим им значения

Для того чтобы выбрать нужный атрибут, в поле атрибут начинаем вводить его название.

4. Нажимаем Сохранить. Видим результат на витрине.

Создание опций​

Опции это некий вид дополнительного товара, услуги или предоставлении выбора покупателю , например, размера товара, цвет итд.


Для создании опции необходимо выполнить следующие действия:

1. Создать группу атрибутов. Меню - Каталог - Опции

1.1. Нажимаем Добавить и переходим к заполнению данными

2. Вводим название опции, ее тип и порядковый номер. Далее ниже нажимаем на кнопку Добавить и добавляем значения опция


Заметка! Опция Файл - это такой вид опций, когда пользователь может в карточке товара загрузить свой файл.
Данный файл, администратор может посмотреть в Меню - Инструменты - Внешние загрузки

3. Переходим в редактирование товара. Меню - Каталог - Товары

3.1. Выбираем вкладку Опции и заполняем данными

Для того чтобы выбрать нужную опцию, в поле Опции начинаем вводить его название.

4. Нажимаем Сохранить и проверяем результат на витрине.

Добавление фильтров товара

1. Включаем модуль фильтра в Модулях. Для этого переходим в раздел модули Меню - Модули - Модули и активируем, включаем модуль Фильтр.

2. В макетах добавляем модуль на макет категорий ()

3.Создаем группу фильтров и фильтры. Меню - Каталог - Фильтры, добавляем группу фильтров и сами фильтры.

3.1 Задаем название группы и Добавляем фильтры.

3.2 Нажимаем сохранить и видим нашу группу.

4. Добавляем фильтры к товару. Меню - Каталог - Товары, на вкладке Связи в поле Фильтр добавляем фильтры. Для добавления фильтра достаточно ввести название фильтра и мышью затем выбрать его.

5. Добавляем фильтры в категорию. Меню - Каталог - Категории на вкладке Данные вводим те фильтры, которые должны отображаться в данной категории. Если не выбрать нужные фильтры, то в данной категории товар не будет фильтроваться и отображаться в фильтре для фильтрации.

6. После данных манипуляций мы сможем увидеть результат работы.

Обратите внимание, что фильтр работает только в категориях.
Для поиска из страниц товаров или производителей, а тем более главной страницы, штатный модуль не предназначен и работать не будет.

Добавление Производителей​

Для добавления производителя необходимо выполнить ниже указанные действия.

1. Переходим в Меню - Каталог - Производители и нажимаем Добавить

2. Далее заполняем поля, такие как Название и при необходимости SEO URL, Изображение и порядок сортировки.

3. Результат мы можем увидеть в списке производителей (по пути http://ваш-сайт.ру/index.php?route=product/manufacturer), а так же при добавлении товара в списке выбора производителя.

4. Далее назначаем производителя, товару. Для этого открываем нужный товар и на вкладке Связи, заполняем поле Производители

Создание информационных статей​

Для создания информационной статьи необходимо перейти в пункт меню Меню - Каталог - Статьи и нажать Добавить

Далее заполняем обязательные поля, такие как, Название статьи, Описание и Мета-тег Title.

Переходим на вкладку Данные и при необходимости меняем настройки:
SEO URL - человеко понятный адрес , например o-nas или informacia
Отображать снизу - Позволяет отображать (или нет) ссылку на статью в подвале сайта
Статус - статус статьи (включено или нет)
Порядок сортировки - порядок сортировки статьи


На заметку! Если у вас нет возможности вывести в подвал сайта статью и узнать его адрес, а чпу у вас не включен, то узнать адрес статьи легко!

Например перейти в карту сайта по адресу http://ваш-сайт.ру/index.php?route=information/sitemap
и там вы увидите все свои статьи

Или сформировать адрес вручную.

Ссылка на статью имеет вид
http://ваш-сайт.ру/index.php?route=information/information&information_id=4 , где 4 это id статьи.

Для того чтобы узнать id статьи, достаточно посмотреть на окончание ссылки.

Ссылку видно в браузере при редактировании статьи.

Файлы для скачивания​

Файлы для скачивания (или цифровой товар) это такие файлы, которые доступны после покупки товара.

Данные файлы будут доступны в личном кабинете покупателя в том случаи, если заказ перешел в статус "Завершено".

Для добавления Файлов для скачивания, сначала необходимо их добавить, а только потом их назначить товару.

1. Переходим в Меню - Каталог - Загрузки и нажимаем Добавить

2. В открывшемся окне заполняем поле Название и загружаем файл.

3. После того как вы добавили все необходимые файлы, вы можете назначить файлы товару.

Для этого открываем нужный товар и на вкладке Связи, заполняем поле Загрузки

Настройка доставки​

Для настройки доставки необходимо перейти в Меню - Модули - Доставка, далее в модуле нажимаем Активировать или Редактировать

В появившемся окне редактирования, производим настройки.

Если модуль доставки только Активировали, не забываем поменять статус на Включено

Класс налога - данное поле отвечает за принадлежность модуля доставки в налоговому классу.
Например, если в модуле доставке выбран класс налога "Налог 18%" то данный способ доставки не будет доступен при оформлении доставки товара, у которого выбран класс налога "Налог 15%".

Если не выбирать ни один из классов налога, то настройка распространяется для всех налоговых классов.

Географическая зона - Данное поле отвечает за доступность метода доставки для определенных регионов.
Например, если в настройках доставки выбрать зону, например Казахстан, а покупатель из Росиии, то данный метод доставки не будет доступен покупателю.

Состояние на складе, статус заказа​

Состояние на складе
Состояний на складе - это статус для товара, который применяется при его отсутствии на складе. Какой именно статус будет применяться к товару, вы можете самостоятельно выбрать на вкладке редактирования товара, поле Отсутствие на складе.
Для того чтобы добавить новое состояние склада, вам необходимо перейти в Меню - Система - Локализация - Состояние на складе и нажать Добавить. Далее ввести название состояния и нажать сохранить.

Статус заказа
После того ка клиент оформил заказ, ему присваивается статус Ожидание. Можно выбрать и другой статус по умолчанию, для этого необходимо перейти в Меню - Система - Настройки и на вкладке Опции выбрать для поля Статус заказа нужный статус.
При оформлении заказа с использование модулей оплаты, данный статус в настройках модуля тоже можно выбрать.
Для добавления новых статусов заказа, необходимо его добавить в Меню - Система - Локализация - Статус заказа

Руководство Разработчика

Обновление

Инструкция ниже, предназначена для обновления магазина с одной версии на другую, например с 2.0.1.0 на 2.3.2.0

Для обновления с версии исправлений rs на следующую версию rs в пределах одной версии движка (например с 2.0.1.1 rs3 на 2.0.1.1 rs4), достаточно просто обновить файлы которые указаны в истории изменений. Или скопировать в корень сайта архив с измененными файлами, если таковой имеется (обычно он называется update-rs-to-rs).

Во многих случаях, простое обновление файлов, тоже рекомендовано для обновлении с версии на версию, но об этом предварительно читайте в теме соответствующей версии.

1. СДЕЛАЙТЕ РЕЗЕРВНУЮ КОПИЮ ВАШЕГО МАГАЗИНА И БАЗЫ ДАННЫХ!!

- Резервное копирование базы данных через ваш магазин Меню - Система - Инструменты - Импорт / Экспорт
(в версии < 2.1 Меню - Инструменты - Бекап / Восстановление).

- Желательно резервную копию базы данных делать через программу управления БД - phpmyadmin , которая установлена на любом хостинге. Это вас избавит от возможных проблем с восстановлением БД в случае удаления таблиц.

- Резервное копирование ваших файлов с помощью FTP или с помощью файлового менеджера на хостинге создать zip для всех существующих файлов и папок магазина

ВНИМАНИЕ!

- Лучше всего обновление производить локально т.е на локальном виртуальном сервере, например OpenServer. Это поможет вам увидеть спокойно все ошибки, обойти различные ограничения сервера.

2. Скачать последнюю версию Магазин "Русская сборка" и загрузите ВСЕ новые файлы поверх существующей установки.

3. Запустите http://<ваш-сайт>/install (Заменив <ваш-сайт> на адрес вашего сайта).

4. Вы увидите краткую инструкцию обновления.

5. Нажмите Кнопку "Продолжить". Через несколько минут вы должны увидеть страницу успешного обновления.
- Если вы видите какие-либо ошибки, сделайте скриншот экрана, сохраните текст ошибок и сообщите об этом на форуме.

6. Очистить все файлы cookie в своем браузере.

7. Перейдите в Административную часть и дважды нажмите на клавиатуре Ctrl+F5 для обновления закешированных CSS стилей. Тоже самое можно сделать на витрине магазина.

8. Перейдите в разделе Меню - Система - Пользователи - Группы пользователей и для Главный администратор отметьте все чекбоксы.., далее нажать Сохранить
- Это дает гарантию, что вы предоставили права администратору на все новые модули системы.

9. Перейдите в разделе Меню - Система- Настройки. Проверьте все значения настроек магазина и нажмите кнопку Сохранить даже если вы ничего не меняли.

10. Перейдите в Меню - Модули - Модификаторы. Далее на вкладке Лог нажать кнопку Очистить. А на верхних кнопках, поочередно нажать Очистить, Обновить.
- Тем самым мы гарантировано обновим кеш модификатора и очистим лог модификатора.
- На вкладке Лог, посмотреть наличие ошибок. Если есть, то необходимо их исправить.

11. Другие корректировки, которые могут потребоваться, в зависимости от того, на какую версию вы переходите от и до.
О таких корректировках будет написано в теме соответствующего релиза.
